## eval_queries.json

Seven evaluation queries derived from the research team's sample question set. Structure per query:

- `query_id`: stable identifier used in all eval output files
- `query_text`: the natural language query as a researcher would type it
- `target_section_keywords`: section header patterns where the answer is expected to live
- `required_keywords`: terms that must appear in a relevant chunk (used for keyword-match validation)
- `cross_filing`: whether the query requires evidence from more than one filing
- `notes`: annotation guidance for the human verifier
- `relevance_grades`: empty dict at creation time; populated by human verifier after Task 5 assigns stable chunk IDs

## Human annotation instructions

After Task 5 (chunker) completes and assigns stable chunk IDs:

1. Run `notebooks/03_chunk.ipynb` to produce `output/chunks_manifest.csv`.
2. For each query in `eval_queries.json`, open the relevant filing(s) in a browser.
3. Identify chunks in `chunks_manifest.csv` that contain the answer.
4. Assign relevance grades using this scale:
   - 0: not relevant
   - 1: marginally relevant (mentions the topic but does not contain the answer)
   - 2: relevant (contains partial answer or supporting context)
   - 3: highly relevant (contains the direct answer with exact figures or verbatim policy text)
5. Populate the `relevance_grades` dict as `{chunk_id: grade}`.
6. Commit the updated `eval_queries.json` with message `docs(fixtures): annotate relevance grades for M0 eval`.

This annotation commit is the gate that unlocks Task 8 (evaluation notebook). The IDE agent must not proceed to Task 8 until this commit exists.
